Development and Production Forms

LapysJS renders scripting files that are tailored for both beginner-level developers to have a better time with JavaScript as a language and pro-level programmers to enjoy a more easing and flexible feel to the language.
LapysJS gives you two forms of its JavaScript once downloaded: The compressed and minified form for use on production sites facing the web. You can find it in the src folder as the lapys.min.js file. and the default and manually coded form for use in development sites. You can also find it in the src folder as lapys.js.

Strict JavaScript Rules

It is suggested at the user’s will to modify the content of the file to nullify or enhance any strict rules the library declares.

LapysCSS

The lapys.js and lapys.min.js files work in tandem with the lapys.css and lapys.min.css files for some of its plug-ins because those its plug-ins are based off animation, dynamism & visual styles on a web page.

Design is not Functionality

The lapys.css and lapys.min.css files do not need the lapys.js and lapys.min.js files because they are functionally independent and focus and design & style only.

bool Function

This is one of LapysJS’ core object functions:
the bool function returns the Boolean object wrapper with its given arguments for a boolean value.

Examples
/* Convert primitive values to their primitive base. */ let bool = function bool(value) { ... }; bool(0); // returns bool('1') // returns
Description

The value passed in the first parameter of the function is converted to a boolean value (if necessary).
If the value is 0, -0, null, false, NaN, undefined, an empty string ("") or is not given, the return value becomes false.
Just as its sibling wrapper object function Boolean, if the parameter is the DOM object document.all, the return value becomes false. Any other value, as well as the string "false" are parsed to return true.

Primitive and Return Values

Do not confuse the primitive Boolean values true and false with the return values of the bool function.
Any object whose value is not null or undefined (including Boolean objects with a false value) are parsed to true when passed in a conditional statement or another bool function.

check Function

In the theory of possibility and statistics, a Bernoulli trial allows for experiments with only two outcomes, a 'success' and 'failure' result.
The check function was inspired by this binomial trial such that when called, the function has an experiment (a condition) it runs constantly (checking) for if the outcome will be a 'success' or a 'failure'.

/* Evaluate if an outcome is 'true' or 'false' in `real-time`. */ check(function condition() { $t$/* $t$$t$Constantly evaluate to 'true' or 'false' the condition below... $t$$t$This uses animation frames to do this in seamless succession. $t$*/ $t$return users > 1000 }, function onsuccess() { $t$/* $t$$t$If the condition is 'true', this function is called. $t$$t$Once this function is called, the constant evaluation is stopped. $t$*/ $t$log('Yes, we made it to 1000 monthly active users!') }, function onfail() { $t$/* If the condition is 'false', this function is called. */ $t$log('Don`t worry, we`ll still make it someday...') }); /* Evaluate if an outcome is 'true' or 'false' at the moment of execution. */ check(true, function() { log('True!') }, function() { log('False!') })

Dynamic Conditional Statement I

The function takes on the form of a if, else if, else control block but constantly evaluates its condition until the condition becomes (or returns) true.

Dynamic Conditional Statement II

The check function was initially meant to be a new JavaScript keyword but as the scripting language does not allow creation of new keywords within its scope, it was just an idea.

check users > 1000 ? $t$log('Yes, we made it to 1000 monthly active users!') : $t$log('Don`t worry, we`ll still make it someday...')

compare Function

The compare function determines whether two values are the same value.
Two values are the same if one of the following holds: both are false. both are null. both are numbers. both are +0. both are -0. both are NaN. both are non-zero and both not NaN and both have equal value. both are strings of equal length and equal order. both are true. both are undefined.

Examples
/* $t$Comparing a wide range of values. $t$Just for fun, you could try experimenting with the '==' and '===' operators to see what $t$$t$values they return instead. */ compare('Hello', 'Hello'); // returns compare(global, window); // returns compare('Hello', 'World'); // returns compare([], []); // returns let obj = {name: 'Lapys', age: 21}; compare(obj, {name: 'Lapys', age: 21}); // returns compare(obj, obj); // returns compare(null, null); // returns /* Special Cases... */ compare(0, -0); // returns compare(-0, -0); // returns compare(NaN, 0 ÷ 0); // returns

Compare is not ‘==’

This comparison is not similar to utilizing the == operator because the == applies other rules to its comparisons and more flexibility between what can be on both sides before parsing (such as "" == false returning true), but compare does not alternate either compared value.

Compare is not ‘===’

The compare function also is not the same as the === operator because the === operator (and the == operator as well) parse the numbers -0 & +0 as equal and the values NaN & Number.NaN as different.
